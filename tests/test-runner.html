Manual Test Cases

TEST 1: Window Creation

Steps:

1. Clear localStorage: DevTools → Application → Local Storage → Clear

2. Refresh the page

Expected: 3 default windows appear (JIT Code Editor, Live Clock, Sticky Notes)

Verify: window.desktopOS.windows.length === 3

TEST 2: Window Minimize

Steps:

1. Click the − button on any window's title bar

Expected: Window disappears from desktop but remains in taskbar

Verify: Window gets 'minimized' class; clicking taskbar item restores it

TEST 3: Window Close

Steps:

1. Click the ✕ button on any window's title bar

Expected: Window is completely removed from DOM and taskbar

Verify: window.desktopOS.windows.length decreases

TEST 4: Window Drag/Move

Steps:

1. Click and hold on window header (not on buttons)

2. Drag mouse to new position

3. Release mouse

Expected: Window follows mouse, stays within screen bounds

Verify: Check window.desktopOS.windows[0].frame.x and .frame.y change

TEST 5: Window Resize

Steps:

1. Locate resize handle (bottom-right corner, resize cursor)

2. Click and drag the handle

Expected: Window resizes, respects minimum size (300x240)

Verify: Check window.desktopOS.windows[0].frame.width and .frame.height

TEST 6: Bring to Front

Steps:

1. Open multiple windows

2. Click on a window that's behind another

Expected: Clicked window becomes topmost with blue glow border

Verify: window.desktopOS.activeWindowId updates; other windows go behind

TEST 7: State Persistence

Steps:

1. Move and resize windows

2. Refresh the page (F5)

Expected: Windows appear at their last positions/sizes

Verify: Check localStorage 'webDesktopState' key contains saved positions

TEST 8: Start Menu

Steps:

1. Click the 🚀 Start button

2. Click elsewhere on desktop

Expected: Menu opens on click, closes on click outside or Escape

Verify: window.desktopOS.startMenuOpen toggles

TEST 9: Desktop Icons

Steps:

1. Double-click on a desktop icon

2. Right-click → Delete → Confirm

Expected: Double-click opens window; right-click deletes icon

Verify: Icons stored in localStorage 'webDesktopIcons'

TEST 10: Widget Lifecycle

Steps:

1. Open a Clock window

2. Minimize the Clock window

3. Wait 5 seconds

4. Restore the window

Expected: Clock shows correct time (not jumped ahead) - timer was cleaned up

Verify: viewWillDisappear() clears interval; no memory leaks

API Reference

// Access the DesktopOS instance
window.desktopOS

// Windows array
window.desktopOS.windows           // Array of WindowController instances

// Methods
window.desktopOS.addWindow(title, widgetType, extraData, x, y, w, h)
window.desktopOS.closeWindow(id)
window.desktopOS.minimizeWindow(id)
window.desktopOS.restoreWindow(id)
window.desktopOS.bringToFront(id)
window.desktopOS.saveState()
window.desktopOS.saveDebounced()

// Window properties
win.windowId                      // Unique ID
win.title                         // Window title
win.widgetType                    // 'clock', 'notes', 'codeEditor', etc.
win.frame                         // {x, y, width, height}
win.bounds                        // {x: 0, y: 0, width, height}
win.center                        // {x, y} (center point)
win.view.zIndex                   // Z-index for layering
win.isMinimized                   // Boolean
win.view.element                  // DOM element

// Lifecycle methods (on widgets)
widget.viewDidLoad()              // Called when view is created
widget.viewWillAppear()           // Called before view appears
widget.viewDidAppear()            // Called after view appears
widget.viewWillDisappear()        // Called before view hides
widget.viewDidDisappear()         // Called after view hides

window.desktopOS.windows

window.desktopOS.windows[0]

localStorage.getItem('webDesktopState')
